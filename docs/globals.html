renderComplete

  • renderComplete(): Promise<void>
    • Defined in queue.ts:76

    Returns a promise which will be resolved after rendered all the components in micro task queues. Note that it was called after onRenderComplete. So if you are implementing a common component, using onRenderComplete would be better. Please don't call await renderComplete() for two times, The second one will be called in a new requestAnimationFrame and browser will render before it.

    Returns Promise<void>
